# Language Guide

## Syfte

Detta dokument beskriver hur Egenkontroll App ska låta.

Appen ska använda enkel, tydlig svenska som fungerar i byggbranschens vardag.

## Ansvar

Detta dokument ansvarar för:

- ton
- tilltal
- skrivregler
- ordnivå
- vad appen ska undvika språkligt

## Hur dokumentet används

Använd detta dokument när:

- UI-texter skrivs
- hjälptexter skapas
- felmeddelanden formuleras
- exporttexter tas fram
- källmaterial omformuleras till appens språk

## Hur dokumentet ska uppdateras

Uppdatera när:

- ny språklig princip beslutas
- användartest visar att text är otydlig
- sakkunnig granskning föreslår bättre vardagsspråk
- projektet hittar uttryck som ska undvikas

## Relation till övriga dokument

- `TERMINOLOGY.md` anger vilka termer som ska användas.
- `DEFINITIONS.md` anger vad termerna betyder.
- `CONTENT_GUIDE.md` beskriver hur texttyper ska skrivas.
- `UX_GUIDELINES.md` beskriver hur språk stödjer användarflödet.

## Grundprinciper

- Skriv som en erfaren arbetsledare skulle förklara arbetet.
- Använd byggbranschens vardagsspråk.
- Var konkret.
- Var kortfattad.
- Skriv vad användaren ska göra härnäst.
- Undvik konsultspråk.
- Undvik myndighetsspråk.
- Undvik ISO-jargong om den inte behövs.
