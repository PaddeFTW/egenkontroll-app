# Changelog

## Syfte

Detta dokument beskriver viktiga ändringar i projektets dokumentation och riktning.

Det är inte en teknisk release notes-fil för kod.

## Hur dokumentet används

Använd detta dokument för att snabbt se:

- när dokumentationsstrukturen ändrades
- vilka styrdokument som lades till
- vilka större produkt- eller dokumentationsändringar som gjorts
- var mer detaljerade beslut finns

Detaljerade beslut dokumenteras i `DECISIONS.md`.

## Format

Varje post ska innehålla:

- datum
- typ av ändring
- kort beskrivning
- berörda dokument

## Ändringar

### 2026-07-04 – Dokumentationsgrund etablerad

**Typ:** Dokumentationsstruktur  
**Beskrivning:** Projektet fick en strukturerad dokumentationsgrund för status, kunskap, domänmodell, terminologi, språk, innehåll, UX, beslut, analys och källmaterial.  
**Berörda dokument:** `README.md`, `docs/APP_BLUEPRINT.md`, `docs/PROJECT_STATUS.md`, `docs/source-material/README.md`, övriga dokumentmallar i `docs/`.
