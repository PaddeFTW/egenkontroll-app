# Analysis

## Syfte

Denna mapp används för framtida analyser av källmaterial, produktfrågor, användarflöden och dokumentationsbeslut.

Analys är arbetsmaterial. Styrande slutsatser ska flyttas till rätt dokument i `docs/`.

## Vad som hör hemma här

- analys av Word-original
- analys av PDF-layout och exportbehov
- genomgång av kontrollpunkter
- jämförelse mellan källmaterial och appens blueprint
- UX-observationer
- öppna frågor inför beslut
- sammanfattningar från arbetsmöten eller granskningssessioner

## Vad som inte hör hemma här

- slutliga produktkrav
- beslut som påverkar scope
- godkänd terminologi
- färdiga UX-riktlinjer
- implementation eller kod

## Hur analys blir styrande dokumentation

1. Gör analysen här.
2. Identifiera verifierade slutsatser.
3. Flytta slutsatserna till rätt dokument:
   - Produktkrav: `../APP_BLUEPRINT.md`
   - Status och nästa steg: `../PROJECT_STATUS.md`
   - Kunskap: `../KNOWLEDGE_BASE.md`
   - Begrepp och modell: `../DOMAIN_MODEL.md`
   - Språk: `../LANGUAGE_GUIDE.md`
   - UX: `../UX_GUIDELINES.md`
   - Beslut: `../DECISIONS.md`
4. Lämna analysen kvar som spårbar bakgrund om den är relevant.
